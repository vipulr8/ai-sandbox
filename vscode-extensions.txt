# Single source of truth for VS Code extensions baked into the ai-sandbox image.
#
# Format:
#   - One `publisher.extension[@version]` ID per line.
#   - Lines starting with `#` and blank lines are ignored.
#   - Build-time env vars are expanded (currently: ${CLAUDE_VERSION}).
#
# Routing in scripts/install-vscode-extensions.sh:
#   - No `@version`        → CLI install (resolves to latest)
#   - `@latest`            → CLI install (resolves to latest)
#   - `@<specific-version>` → direct VSIX download from the marketplace asset URL
#
# NOTE: anthropic.claude-code is NOT baked here. It's installed post-attach by
# dev.sh via `code-server --install-extension --force` against the user's running
# VS Code Server, because that path triggers the extension's install hook and
# initializes its auth state correctly. Bake-time install skips that hook and
# leaves API-mode auth broken.
